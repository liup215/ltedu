# LTEDU API

## Overview

This is the backend API for the LTEDU platform. It is a high-performance service written in Go, providing all the necessary data and business logic for the LTEDU web and mobile applications.

## Recent Progress

- **Unified Verification Logic:** The backend now uses a central service for sending verification codes to both email and phone targets. The SendCode API validates captcha, determines verification type, and delegates to a unified GenerateAndSendCode method, improving maintainability and extensibility.
- **API Consistency:** All endpoints follow strict versioning and request/response patterns for reliability and integration with the frontend.
- **Frontend Integration:** Features such as image captcha, verification code countdown, and error handling are fully integrated with the frontend application.

## Technology Stack

- **Language**: Go
- **Web Framework**: Gin (assumed)
- **ORM**: GORM with `gorm/gen` for type-safe DAO layer
- **Database**: SQLite (for development), PostgreSQL (for production)
- **Configuration**: Viper (assumed)
- **Containerization**: Docker

## Project Structure

The project follows a standard Go web application structure:

```
.
├── cmd/              # Main application entry point
├── conf/             # Configuration management
├── dao/              # Data Access Objects (generated by gorm/gen)
├── gen/              # GORM generator script
├── lib/              # Shared libraries (database, logging, etc.)
├── model/            # Data models and request/response structs
├── server/           # HTTP server and route setup
├── service/          # Business logic layer
├── sql/              # SQL migration files
├── task/             # Asynchronous tasks and jobs
├── Dockerfile        # Docker build configuration
└── go.mod            # Go module dependencies
```

## Getting Started

### Prerequisites

- Go (version 1.20 or higher)
- Docker and Docker Compose
- Make (optional)

### Running with Docker (Recommended)

The easiest way to get the API running is by using the provided `docker-compose.yml` at the root of the monorepo.

```bash
# From the root ltedu/ directory
docker-compose up -d ltedu-api
```

The API will be available at `http://localhost:9000`.

### Running Locally

1.  **Navigate to the API directory:**
    ```bash
    cd ltedu-api
    ```

2.  **Install dependencies:**
    ```bash
    go mod tidy
    ```

3.  **Configure your environment:**
    Create a `config.yaml` file in the `conf` directory or set environment variables according to the `conf/config.go` specification. The application requires database connection details and other service configurations.

4.  **Run the application:**
    ```bash
    go run ./cmd/main.go
    ```

## API Endpoints

The API follows the design patterns outlined in the project's HTTP rules. All endpoints are versioned under `/api/v1/`.

- **Authentication**: `POST /api/v1/login`
- **CRUD Operations**: All resources use a consistent endpoint structure:
  - `POST /api/v1/{resource}/list`
  - `POST /api/v1/{resource}/byId`
  - `POST /api/v1/{resource}/create`
  - `POST /api/v1/{resource}/edit`
  - `POST /api/v1/{resource}/delete`

Refer to the service and model definitions for detailed request/response formats.
