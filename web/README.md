# A-level Toolkit Web

## Overview

This is the frontend application for the A-level Toolkit platform. It is a modern, responsive single-page application (SPA) built with Vue 3 and TypeScript, providing a rich user interface for students, teachers, and administrators.

## Recent Progress

- **Register.vue Email Verification Code Countdown:** After sending a verification code, the "Send Code" button is disabled and displays a 60-second countdown before re-enabling. This improves registration security and user experience.
- **Runtime Configuration:** The app loads API base URL and other settings from `public/config.json` at runtime, supporting flexible deployment.
- **Strict API Patterns:** All HTTP requests follow unified backend conventions for consistency and maintainability.
- **Frontend and backend integration:** Features such as image captcha, unified verification logic, and error handling are fully integrated.

## Technology Stack

-   **Framework**: Vue 3 (with Composition API and `<script setup>`)
-   **Language**: TypeScript
-   **Build Tool**: Vite
-   **Routing**: Vue Router
-   **State Management**: Pinia
-   **Styling**: Tailwind CSS
-   **HTTP Client**: Axios

## Getting Started

### Prerequisites

-   Node.js (version 18 or higher)
-   npm or yarn
-   Docker and Docker Compose

### Running with Docker (Recommended)

The easiest way to get the web application running is by using the provided `docker-compose.yml` at the root of the monorepo.

```bash
# From the root ltedu/ directory
docker-compose up -d ltedu-web
```

The application will be available at `http://localhost:8080`.

### Running Locally

1.  **Navigate to the web directory:**
    ```bash
    cd ltedu-web
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    ```

3.  **Configure your environment:**
    The application reads its configuration from `public/config.json` at runtime. This file is generated by the `entrypoint.sh` script in the Docker container. For local development, you can create or modify this file to point to your local API server.

    **Example `public/config.json`:**
    ```json
    {
      "API_BASE_URL": "http://localhost:9000"
    }
    ```

4.  **Run the development server:**
    ```bash
    npm run dev
    ```

    The application will be available at `http://localhost:5173` (or another port if 5173 is in use).

## Build for Production

To build the application for production:

```bash
npm run build
```

This will create a `dist` directory with the optimized, static assets ready for deployment.

## Runtime Configuration

The application is designed to be environment-agnostic. Instead of building separate images for different environments (dev, staging, prod), it fetches its configuration from `/config.json` at runtime.

The `entrypoint.sh` script in the Docker container generates this `config.json` file from environment variables passed to the container. This allows for dynamic configuration without rebuilding the application.

**How it works:**
1.  The `entrypoint.sh` script reads environment variables (e.g., `API_BASE_URL`).
2.  It creates `public/config.json` with these values.
3.  The application, on startup, fetches `/config.json` and uses the values to configure the `apiClient` and other parts of the app.

This approach aligns with modern DevOps practices and makes deployment more flexible.
